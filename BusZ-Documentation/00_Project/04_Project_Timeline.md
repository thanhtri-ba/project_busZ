# Project Timeline

**Project:** BusZ - Intercity Bus Ticket Booking Platform

**Version:** 1.0

**Document Type:** Project Timeline

---

# 1. Purpose

Tài liệu này mô tả kế hoạch phát triển dự án BusZ theo từng giai đoạn (Sprint).

Mỗi Sprint sẽ có:

- Mục tiêu
- Module cần hoàn thành
- Deliverables
- Tiêu chí hoàn thành

Timeline này được xây dựng theo phương pháp Agile Scrum.

---

# 2. Development Methodology

Dự án sử dụng Agile Scrum.

Mỗi Sprint kéo dài:

- 2 tuần

Sau mỗi Sprint sẽ có:

- Sprint Review
- Sprint Retrospective
- Planning Sprint tiếp theo

# 14. Risk Management

Các rủi ro:

- Thiếu dữ liệu tuyến xe.
- API thanh toán thay đổi.
- Thiết kế UI thay đổi.
- Database cần mở rộng.
- Chậm tiến độ.

Giải pháp:

- Thiết kế module độc lập.
- Sử dụng Git.
- Review code hàng tuần.
- Viết tài liệu đầy đủ.
- Kiểm thử sau mỗi Sprint.

---

# 15. Success Criteria

Một Sprint được xem là hoàn thành khi:

- Chức năng hoạt động đúng.
- Database đã cập nhật.
- API hoàn chỉnh.
- Flutter kết nối thành công.
- Không còn lỗi nghiêm trọng.
- Có tài liệu cập nhật.
